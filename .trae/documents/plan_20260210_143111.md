# 注册页面功能调整实现计划

## 实现目标
根据角色类型动态显示不同字段，并为短信链接进入的用户设置默认值。

## 实现步骤

### 1. 修改后端用户模型
- 在 `User` 模型中添加两个新字段：
  - `identity`: 字符串类型，可选值为 'applicant'（申请人）和 'respondent'（被申请人）
  - `caseAmount`: 数字类型，用于存储涉案金额
- 确保这些字段对于调解员和管理员角色是可选的

### 2. 修改前端注册页面
- 添加条件渲染逻辑，根据选择的角色类型显示不同字段：
  - 个人用户和企业用户：显示"选择身份"和"涉案金额"字段
  - 调解员和管理员：不显示这些字段
- 修改短信链接验证后的默认值设置：
  - 默认选中"个人用户"角色
  - 默认选中"申请人"身份
- 添加表单验证规则，确保必填字段的完整性

### 3. 修改后端注册控制器
- 更新注册控制器，处理新添加的 `identity` 和 `caseAmount` 字段
- 确保字段验证逻辑正确，根据角色类型处理不同的字段要求

### 4. 测试验证
- 测试不同角色类型的注册流程
- 测试短信链接进入的默认值设置
- 测试表单验证和提交功能

## 技术实现要点

### 前端实现
- 使用 Ant Design 的 Form 组件和条件渲染
- 使用 `useState` 管理表单状态
- 使用 `Form.Item` 的 `shouldUpdate` 属性实现动态字段显示

### 后端实现
- 使用 Mongoose 的 Schema 扩展添加新字段
- 确保字段验证逻辑正确
- 保持 API 响应格式的一致性

## 文件修改清单

### 后端文件
- `/backend/src/models/User.ts` - 添加新字段
- `/backend/src/auth/controller.ts` - 更新注册逻辑

### 前端文件
- `/frontend/src/pages/Register.tsx` - 修改注册页面逻辑和UI

## 预期效果
- 用户注册时，根据选择的角色类型看到不同的字段
- 通过短信链接进入的用户自动设置默认值
- 表单验证逻辑正确，确保数据完整性
- 注册流程顺畅，用户体验良好
