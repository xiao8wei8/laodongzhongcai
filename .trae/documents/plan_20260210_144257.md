# 注册页面修复计划

## 问题分析
1. **角色选择无默认值**：当前注册页面角色选择框没有默认值
2. **角色文本不一致**："个人用户"和"企业用户"需要改为"个人账户"和"企业账户"
3. **条件渲染失效**："选择身份"和"涉案金额"字段没有根据角色选择动态显示/隐藏
4. **字段类型问题**：涉案金额应该使用数字输入框

## 修复方案

### 1. 修改角色选择默认值
- 在Form组件初始化时设置默认角色为"个人用户"
- 确保通过短信链接进入时保持现有默认值逻辑

### 2. 更新角色选项文本
- 将"个人用户"改为"个人账户"
- 将"企业用户"改为"企业账户"
- 保持其他角色文本不变

### 3. 修复条件渲染逻辑
- 优化Form.Item的shouldUpdate属性配置
- 确保只有在选择"个人账户"或"企业账户"时显示"选择身份"和"涉案金额"字段
- 当角色切换时正确处理字段显示/隐藏

### 4. 优化字段类型
- 确保"选择身份"使用下拉选择框
- 确保"涉案金额"使用数字输入框

## 技术实现要点

### 前端修改
- 在Form组件中添加initialValues属性设置默认角色
- 更新Select组件的Option文本
- 调整Form.Item的shouldUpdate逻辑，确保条件渲染正确
- 保持现有验证逻辑不变

### 文件修改
- `/frontend/src/pages/Register.tsx` - 修复注册页面逻辑和UI

## 预期效果
- 注册页面默认选择"个人账户"角色
- 角色选项文本与要求一致
- "选择身份"和"涉案金额"字段根据角色选择动态显示/隐藏
- 表单验证逻辑正确，用户体验良好
