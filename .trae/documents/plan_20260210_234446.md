# 案件分配机制调整方案

## 1. 需求分析

根据要求，需要实现以下案件分配机制：
- **默认规则**: 当日值班调解员自动成为该案件的承办调解员
- **调整权限**: 管理员可根据实际情况，在后台调整案件分配
- **连续性保障**: 确保每个案件有明确的责任调解员

## 2. 当前系统分析

- **Case 模型**: 已有 `mediatorId` 字段用于存储案件的承办调解员
- **分配功能**: 已有 `assignMediator` 函数用于分配调解员，但创建案件时未自动分配
- **权限控制**: 管理员已可访问 `/api/case/:id/mediator` 路由调整案件分配
- **值班机制**: 系统中暂无"值班调解员"的概念

## 3. 实施计划

### 3.1 数据模型调整

1. **修改 User 模型**
   - 添加 `isOnDuty` 字段，用于标记当日值班调解员
   - 添加 `lastOnDutyDate` 字段，用于记录上次值班日期

### 3.2 后端功能实现

1. **添加值班管理功能**
   - 新增 `setOnDutyMediator` 函数，用于设置当日值班调解员
   - 确保同一时间只有一位调解员处于值班状态

2. **修改案件创建功能**
   - 修改 `createCase` 函数，在创建案件时自动查询当日值班调解员
   - 自动将值班调解员分配为案件的承办调解员
   - 确保案件状态正确更新

3. **增强案件分配功能**
   - 确保 `assignMediator` 函数支持管理员调整案件分配
   - 添加日志记录，跟踪案件分配变更

### 3.3 路由配置

1. **添加值班管理路由**
   - 新增 `/api/user/on-duty` 路由，用于设置当日值班调解员
   - 仅允许管理员访问

2. **确保现有路由权限正确**
   - 验证 `/api/case/:id/mediator` 路由权限设置正确

### 3.4 前端功能实现

1. **添加值班管理界面**
   - 在管理员后台添加"值班管理"模块
   - 支持设置和查看当日值班调解员

2. **增强案件管理界面**
   - 在案件详情页面显示当前承办调解员
   - 支持管理员调整案件分配

3. **添加状态提示**
   - 在案件列表和详情页面显示案件分配状态
   - 确保每个案件都有明确的责任调解员

## 4. 技术实现细节

### 4.1 数据模型修改

- **User 模型**: 添加 `isOnDuty` (Boolean) 和 `lastOnDutyDate` (Date) 字段

### 4.2 后端逻辑

- **值班管理**: 实现 `setOnDutyMediator` 函数，确保同一时间只有一位调解员值班
- **案件创建**: 修改 `createCase` 函数，自动查询并分配当日值班调解员
- **权限控制**: 确保管理员可调整案件分配

### 4.3 前端实现

- **值班管理界面**: 在管理员后台添加值班管理模块
- **案件分配界面**: 支持管理员调整案件分配
- **状态提示**: 显示案件分配状态和责任调解员信息

## 5. 验证和测试

- **功能测试**: 测试案件自动分配、手动调整等功能
- **权限测试**: 验证管理员权限和普通用户权限
- **边界测试**: 测试无值班调解员时的处理逻辑
- **集成测试**: 确保与现有系统的集成正常

通过以上调整，将实现案件分配的自动化和灵活性，确保每个案件都有明确的责任调解员，同时支持管理员根据实际情况进行调整。
