# 传统工作场景升级功能测试计划

## 测试目标
验证传统工作场景升级功能的完整流程，确保从调解员登记到访者信息到访客注册的整个过程正常工作。

## 测试环境
- 后端服务器：http://localhost:5002
- 前端服务器：http://localhost:5173
- MongoDB 数据库：本地运行

## 测试步骤

### 1. 登录系统
- 访问前端登录页面：http://localhost:5173/login
- 使用调解员账号登录系统

### 2. 到访登记测试
- 导航到"到访登记"页面
- 填写来访者信息：
  - 姓名：测试用户
  - 电话：13800138000
  - 到访类型：咨询
  - 纠纷类型：劳动报酬
  - 事由摘要：测试事由
- 点击"提交"按钮
- 验证系统是否生成正确格式的登记编号（如 JD20260210001）

### 3. 发送短信链接测试
- 在到访记录列表中找到刚创建的记录
- 点击"发送短信链接"按钮
- 确认对话框中点击"确定"
- 验证系统是否显示"短信链接发送成功"的提示

### 4. 测试加密链接
- 模拟访客收到短信并点击链接
- 链接格式应为：http://localhost:5173/register?token=xxx
- 访问该链接，验证注册页面是否自动填充姓名和电话
- 验证系统是否显示"链接验证成功，已自动填充信息"的提示

### 5. 完成注册测试
- 在注册页面填写剩余信息：
  - 用户名：testuser
  - 密码：123456
  - 邮箱：test@example.com
  - 身份证号：110101199001011234
- 点击"注册"按钮
- 验证系统是否显示"注册成功，请登录"的提示
- 验证是否跳转到登录页面

### 6. 登录测试
- 使用新注册的账号登录系统
- 验证是否登录成功并跳转到首页

## 预期结果
1. 登记编号格式正确：街道代码+年月日+序号
2. 短信链接发送成功
3. 加密链接验证成功并自动填充信息
4. 注册流程完成
5. 新用户登录成功

## 测试工具
- 浏览器：Chrome 或 Firefox
- 开发者工具：用于查看网络请求和控制台错误
- MongoDB Compass：用于查看数据库记录

## 注意事项
- 由于是测试环境，短信服务可能无法实际发送短信，需要通过代码逻辑验证
- 确保网络连接正常
- 确保数据库服务运行正常
