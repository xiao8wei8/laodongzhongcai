# 案件详情页面优化计划

## 一、需求分析

根据参照图和文档要求，需要对当前的案件详情页面进行以下优化：

1. **案件全流程时间轴管理**：
   - 四大阶段可视化展示（案件登记、案件受理、调解进展、结案）
   - 智能记录功能（"记一笔"快捷记录、文件上传与识别、智能日程提醒、强制提醒机制）

2. **线上调解与证据管理**：
   - 线上调解会议室功能
   - 证据材料管理（独立证据材料库、多种格式支持、证据标签分类）

## 二、当前实现分析

当前的 `CaseDetail.tsx` 页面已经实现了：
- 案件基本信息展示
- 案件进度时间轴
- 证据管理（文件上传）
- 状态更新和调解员分配功能

## 三、优化计划

### 1. 增强案件进度时间轴
- **添加四大阶段标识**：在时间轴上明确标识案件登记、案件受理、调解进展、结案四个阶段
- **添加阶段状态指示器**：使用不同颜色和图标表示各阶段的完成状态
- **添加案件进度条**：在页面顶部添加可视化进度条，显示案件整体进展

### 2. 添加智能记录功能
- **"记一笔"快捷记录**：添加文本输入框，允许调解员快速记录调解过程中的关键信息
- **智能日程提醒**：识别记录中的日期信息，询问是否添加到日程提醒
- **强制提醒机制**：实现案件受理后第15天的自动提醒功能

### 3. 优化证据管理
- **证据标签分类**：添加标签输入功能，允许为上传的证据添加分类标签
- **证据材料库组织**：按类型和时间排序展示证据材料
- **多种格式支持**：确保支持图片、PDF、Word、视频等多种格式

### 4. 添加线上调解功能
- **调解会议预约**：添加会议预约功能，生成会议链接
- **会议记录管理**：展示历史调解会议记录
- **电子签名板**：集成电子签名功能，用于在线签署调解协议

### 5. 界面优化
- **布局调整**：参考参照图3，调整页面布局，左侧为时间轴和记录区域，右侧为案件基本信息和证据材料
- **响应式设计**：确保页面在不同设备上都能正常显示
- **交互优化**：添加更多交互反馈，提升用户体验

## 四、技术实现

1. **前端修改**：
   - 修改 `CaseDetail.tsx` 文件，添加新功能和优化界面
   - 使用 Ant Design 组件库实现界面元素
   - 添加必要的状态管理和交互逻辑

2. **后端支持**：
   - 需要后端提供相应的 API 端点，支持新功能的数据操作
   - 特别是"记一笔"记录的保存和获取、证据标签的管理、调解会议的预约等

## 五、预期效果

通过以上优化，案件详情页面将更加符合用户需求，提供更全面的案件管理功能，包括可视化的时间轴管理、智能记录功能、完善的证据管理和线上调解支持，从而提高调解工作的效率和质量。
