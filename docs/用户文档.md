# 劳动仲裁调解系统 用户文档

## 1. 系统简介

### 1.1 系统概述

劳动仲裁调解系统是一个专为劳动仲裁机构设计的数字化平台，旨在简化调解流程、提高工作效率、保障当事人合法权益。系统支持多角色访问，包括管理员、调解员、个人用户和企业用户，提供案件管理、到访登记、调解申请、工作台统计等核心功能。

### 1.2 系统特点

- **多角色支持**: 管理员、调解员、个人用户、企业用户
- **全流程管理**: 从案件申请到调解完成的全流程跟踪
- **实时数据统计**: 工作台实时展示案件状态和统计数据
- **智能案件分配**: 基于调解员负载和专业领域的智能分配
- **便捷到访登记**: 快速记录和管理到访信息
- **广播消息通知**: 重要信息及时送达所有用户
- **安全可靠**: 采用 JWT 认证和数据加密技术

### 1.3 适用范围

- **劳动仲裁机构**: 处理劳动争议案件
- **企业和个人**: 申请劳动争议调解
- **调解员**: 处理和跟踪调解案件
- **管理员**: 系统管理和权限控制

## 2. 系统架构

### 2.1 技术架构

- **前端**: React 18 + TypeScript 5 + Ant Design 5
- **后端**: Node.js + Express + MongoDB + Redis
- **认证**: JWT (JSON Web Token)
- **实时通信**: Socket.io
- **数据可视化**: ECharts + Recharts

### 2.2 系统模块

1. **认证授权模块**: 用户注册、登录、权限管理
2. **工作台模块**: 统计数据、待办事项、快捷操作
3. **案件管理模块**: 案件创建、查询、状态更新
4. **到访登记模块**: 访客信息登记、记录管理
5. **调解申请模块**: 申请创建、审批、进度跟踪
6. **广播消息模块**: 消息发布、接收、管理
7. **数据分析模块**: 案件趋势、统计报表

### 2.3 数据流程

1. **用户认证**: 用户登录 → 系统验证 → 生成 JWT 令牌
2. **案件申请**: 用户提交申请 → 系统验证 → 创建案件记录
3. **调解流程**: 案件分配 → 调解员处理 → 状态更新 → 调解完成
4. **到访登记**: 访客信息录入 → 系统存储 → 统计分析
5. **消息通知**: 管理员发布 → 系统广播 → 用户接收

### 3.3 登录步骤

1. **打开登录页面**
   - 在浏览器中输入系统地址，进入登录页面

2. **选择用户角色**
   - 从下拉菜单中选择您的角色：管理员、调解员、个人用户或企业用户

   - **用户名**: 您的系统用户名

4. **点击登录按钮**
   - 系统验证您的身份信息
   - 验证成功后，根据您的角色跳转到相应的工作台

### 3.4 登录问题排查

- **用户名或密码错误**: 请检查您输入的用户名和密码是否正确
- **角色选择错误**: 请确保选择了正确的用户角色
- **账号被锁定**: 连续多次登录失败可能导致账号被临时锁定，请联系管理员
- **网络连接问题**: 请检查您的网络连接是否正常

### 3.5 首次登录

- **新用户注册**: 点击登录页面下方的"注册"链接，填写注册信息
- **初始密码**: 首次登录时，系统会提示您修改初始密码
- **个人信息完善**: 登录后请及时完善个人信息，如联系方式、所属部门等

### 3.6 权限说明

系统基于角色的权限控制，不同角色拥有不同的功能权限和可访问页面：

#### 3.6.1 角色权限对比

| 权限项 | 管理员 | 调解员 | 个人用户 | 企业用户 |
|--------|--------|--------|----------|----------|
| **控制台访问** | ✓ | ✓ | ✓ | ✓ |
| **案件查询** | ✓ | ✓ | ✓ | ✓ |
| **案件详情查看** | ✓ | ✓ | ✓ | ✓ |
| **申请仲裁** | ✗ | ✗ | ✓ | ✓ |
| **来访登记** | ✗ | ✓ | ✗ | ✗ |
| **发布通知公告** | ✓ | ✓ | ✗ | ✗ |
| **查看数据分析** | ✓ | ✓ | ✗ | ✗ |
| **用户管理** | ✓ | ✗ | ✗ | ✗ |
| **系统设置** | ✓ | ✗ | ✗ | ✗ |

#### 3.6.2 角色功能说明

- **管理员 (admin)**:
  - 拥有系统所有权限
  - 可管理用户账号和系统配置
  - 可发布通知公告和查看数据分析
  - 可访问所有页面和功能

- **调解员 (mediator)**:
  - 可处理和跟踪案件
  - 可登记来访信息
  - 可发布通知公告
  - 可查看数据分析
  - 不能管理用户和系统设置

- **个人用户 (personal)**:
  - 可查询个人案件
  - 可申请劳动争议调解
  - 不能访问管理功能
  - 不能查看其他用户的案件

- **企业用户 (company)**:
  - 可查询企业案件
  - 可申请劳动争议调解
  - 不能访问管理功能
  - 不能查看其他企业的案件

## 9. 系统操作流程

### 9.1 案件处理流程

1. **案件申请**
   - 个人或企业用户提交调解申请
   - 系统验证申请信息，生成案件编号

2. **案件受理**
   - 调解员查看申请详情
   - 确认受理案件，分配处理人员

3. **调解准备**
   - 收集案件相关材料
   - 了解双方当事人的诉求和争议焦点
   - 制定调解方案

4. **调解过程**
   - 组织双方当事人进行调解
   - 记录调解过程和沟通内容
   - 协助双方达成共识

5. **案件结案**
   - 填写调解结果
   - 上传相关材料和证明文件
   - 双方确认调解结果
   - 系统归档案件

### 9.2 到访登记流程

1. **访客到达**
   - 访客到达仲裁机构
   - 前台接待人员进行初步了解

2. **信息登记**
   - 填写访客基本信息
   - 记录到访类型和原因
   - 选择处理方式

3. **分配处理**
   - 系统根据到访原因和纠纷类型分配调解员
   - 通知相关调解员处理

4. **到访处理**
   - 调解员与访客沟通
   - 了解具体情况，提供初步建议
   - 记录处理结果

5. **后续跟踪**
   - 对需要进一步处理的到访进行跟踪
   - 定期回访，了解处理进展

### 9.3 调解申请流程

1. **申请提交**
   - 用户在线填写调解申请
   - 上传相关证据材料
   - 提交申请等待审批

2. **申请审批**
   - 调解员查看申请详情
   - 验证申请材料的完整性
   - 决定是否受理申请

3. **申请处理**
   - 对受理的申请进行处理
   - 安排调解时间和地点
   - 通知双方当事人

4. **调解过程**
   - 按照案件处理流程进行调解
   - 记录调解过程和结果

5. **申请结案**
   - 填写申请处理结果
   - 上传相关材料
   - 完成申请处理

## 11. 常见问题与解决方案

### 11.1 登录问题

- **忘记密码**
  - 点击登录页面的"忘记密码"链接
  - 输入注册时的邮箱
  - 点击邮箱中的重置链接，设置新密码

- **账号被锁定**
  - 连续多次登录失败导致账号被锁定
  - 等待30分钟后自动解锁
  - 或联系管理员手动解锁

- **无法登录系统**
  - 检查网络连接是否正常
  - 确认用户名和密码是否正确
  - 确认用户角色选择是否正确
  - 联系管理员检查账号状态

### 11.2 功能使用问题

- **无法创建案件**
  - 检查是否有创建案件的权限
  - 确认填写的信息是否完整
  - 检查网络连接是否正常

- **案件状态不更新**
  - 检查是否有更新案件状态的权限
  - 确认网络连接是否正常
  - 刷新页面后重试

- **无法上传文件**
  - 检查文件大小是否超过限制（最大10MB）
  - 确认文件格式是否支持
  - 检查网络连接是否正常

- **消息不显示**
  - 刷新页面后重试
  - 检查消息通知设置是否正确
  - 联系管理员检查系统消息服务

### 11.3 系统异常问题

- **页面加载缓慢**
  - 检查网络连接是否正常
  - 关闭浏览器中不必要的标签页
  - 清除浏览器缓存后重试

- **系统报错**
  - 记录错误信息
  - 刷新页面后重试
  - 联系管理员报告问题

- **数据丢失**
  - 立即联系管理员
  - 提供操作时间和具体情况
  - 系统会定期备份数据，可从备份恢复

## 15. 术语解释

### 15.1 系统术语

- **案件**: 劳动争议调解的基本单位，包含完整的争议信息和处理过程
- **调解**: 第三方居中协调，帮助当事人解决劳动争议的过程
- **申请人**: 提起劳动争议调解申请的一方
- **被申请人**: 劳动争议调解申请的相对方
- **调解员**: 负责处理劳动争议调解案件的专业人员
- **仲裁员**: 劳动仲裁机构中具有仲裁资格的人员
- **到访**: 当事人到劳动仲裁机构咨询或处理劳动争议的行为
- **调解协议**: 当事人在调解过程中达成的书面协议

### 15.2 操作术语

- **受理**: 调解员确认接受并开始处理案件
- **立案**: 系统正式登记案件，生成案件编号
- **结案**: 案件处理完成，系统归档案件
- **撤诉**: 申请人撤回调解申请
- **转仲裁**: 调解不成，案件转为仲裁程序
- **和解**: 当事人自行达成和解协议
- **调解成功**: 当事人在调解员的帮助下达成调解协议
- **调解失败**: 当事人未能达成调解协议

### 15.3 状态术语

- **pending**: 待处理，案件或申请等待处理
- **processing**: 处理中，案件或申请正在处理
- **completed**: 已完成，案件或申请处理完毕
- **failed**: 失败，案件或申请处理失败
- **accepted**: 已接受，申请被接受
- **rejected**: 已拒绝，申请被拒绝
- **withdrawn**: 已撤回，申请被撤回
