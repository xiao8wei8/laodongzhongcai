# 劳动仲裁调解系统 运维文档

## 1. 系统架构

### 1.1 技术架构

- **前端**: React 18 + TypeScript 5 + Ant Design 5
- **后端**: Node.js + Express + MongoDB + Redis
- **认证**: JWT (JSON Web Token)
- **实时通信**: Socket.io
- **数据可视化**: ECharts + Recharts

### 1.2 系统模块

1. **认证授权模块**: 用户注册、登录、权限管理
2. **工作台模块**: 统计数据、待办事项、快捷操作
3. **案件管理模块**: 案件创建、查询、状态更新
4. **到访登记模块**: 访客信息登记、记录管理
5. **调解申请模块**: 申请创建、审批、进度跟踪
6. **广播消息模块**: 消息发布、接收、管理
7. **数据分析模块**: 案件趋势、统计报表

### 1.3 依赖关系

- **前端依赖**: React、TypeScript、Ant Design、React Router、Zustand、Axios、ECharts、Recharts
- **后端依赖**: Express、Mongoose、JSONWebToken、bcryptjs、CORS、Dotenv、Socket.io、Redis、Express-validator
- **数据库依赖**: MongoDB 4.0+、Redis 5.0+
- **系统依赖**: Node.js 14+、npm 6+

## 2. 系统部署

### 2.1 环境准备

#### 2.1.1 服务器要求

- **硬件配置**:
  - CPU: 4核以上
  - 内存: 8GB以上
  - 存储: 100GB以上 SSD
  - 网络: 100Mbps以上带宽

- **操作系统**:
  - Linux: Ubuntu 20.04 LTS 或 CentOS 7+
  - Windows: Windows Server 2016 或更高版本
  - macOS: macOS 10.15 或更高版本

- **软件依赖**:
  - Node.js 14.17.0 或更高版本
  - npm 6.14.13 或更高版本
  - MongoDB 4.4.0 或更高版本
  - Redis 5.0.0 或更高版本
  - Nginx 1.18.0 或更高版本（可选，用于反向代理）

#### 2.1.2 网络配置

- **防火墙设置**:
  - 开放端口: 5002 (后端API)、5173 (前端开发服务器)
  - 生产环境: 仅开放 80 (HTTP) 和 443 (HTTPS) 端口

- **域名配置**:
  - 前端域名: `example.com` 或 `www.example.com`
  - 后端API域名: `api.example.com`

- **SSL证书**:
  - 生产环境必须使用 HTTPS
  - 推荐使用 Let's Encrypt 免费证书

### 2.2 部署步骤

#### 2.2.1 前端部署

1. **克隆代码**
   ```bash
   git clone <repository-url> /opt/laodongzhongcai
   cd /opt/laodongzhongcai/frontend
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **构建项目**
   ```bash
   npm run build
   ```

4. **部署静态文件**
   - 构建产物位于 `dist` 目录
   - 使用 Nginx 或 Apache 部署静态文件
   - 配置示例 (Nginx):
     ```nginx
     server {
       listen 80;
       server_name example.com;
       return 301 https://$host$request_uri;
     }

     server {
       listen 443 ssl;
       server_name example.com;

       ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
       ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

       root /opt/laodongzhongcai/frontend/dist;
       index index.html;

       location / {
         try_files $uri $uri/ /index.html;
       }

       location /api {
         proxy_pass http://localhost:5002;
         proxy_http_version 1.1;
         proxy_set_header Upgrade $http_upgrade;
         proxy_set_header Connection 'upgrade';
         proxy_set_header Host $host;
         proxy_cache_bypass $http_upgrade;
       }
     }
     ```

#### 2.2.2 后端部署

1. **进入后端目录**
   ```bash
   cd /opt/laodongzhongcai/backend
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **配置环境变量**
   - 复制 `.env.example` 文件为 `.env.production`
   - 编辑 `.env.production` 文件，填写相应配置
   ```bash
   # 服务器配置
   PORT=5002
   NODE_ENV=production

   # 数据库配置
   MONGO_URI=mongodb://localhost:27017/laodong_production
   REDIS_URL=redis://localhost:6379

   # JWT配置
   JWT_SECRET=your_production_jwt_secret_key
   JWT_EXPIRES_IN=24h

   # 文件存储配置
   FILE_UPLOAD_PATH=./public/uploads
   MAX_FILE_SIZE=10485760
   ```

4. **构建项目**
   ```bash
   npm run build
   ```

5. **启动服务**
   - 使用 PM2 管理进程
   ```bash
   # 安装 PM2
   npm install -g pm2

   # 启动服务
   pm2 start dist/server.js --name laodongzhongcai-backend

   # 设置开机自启
   pm2 startup
   pm2 save
   ```

#### 2.2.3 数据库部署

1. **MongoDB 部署**
   - **Ubuntu**:
     ```bash
     # 安装 MongoDB
     wget -qO - https://www.mongodb.org/static/pgp/server-4.4.asc | sudo apt-key add -
     echo "deb [ arch=amd64,arm64 ] https://repo.mongodb.org/apt/ubuntu focal/mongodb-org/4.4 multiverse" | sudo tee /etc/apt/sources.list.d/mongodb-org-4.4.list
     sudo apt-get update
     sudo apt-get install -y mongodb-org

     # 启动 MongoDB
     sudo systemctl start mongod
     sudo systemctl enable mongod
     ```

   - **CentOS**:
     ```bash
     # 安装 MongoDB
     sudo yum install -y mongodb-org

     # 启动 MongoDB
     sudo systemctl start mongod
     sudo systemctl enable mongod
     ```

   - **创建数据库**:
     ```bash
     mongo
     > use laodong_production
     > db.createUser({
         user: "laodong",
         pwd: "your_password",
         roles: [{ role: "readWrite", db: "laodong_production" }]
       })
     > exit
     ```

2. **Redis 部署**
   - **Ubuntu**:
     ```bash
     # 安装 Redis
     sudo apt-get update
     sudo apt-get install -y redis-server

     # 启动 Redis
     sudo systemctl start redis-server
     sudo systemctl enable redis-server
     ```

   - **CentOS**:
     ```bash
     # 安装 Redis
     sudo yum install -y epel-release
     sudo yum install -y redis

     # 启动 Redis
     sudo systemctl start redis
     sudo systemctl enable redis
     ```

   - **配置 Redis**:
     ```bash
     sudo nano /etc/redis/redis.conf
     # 修改 bind 为 127.0.0.1
     # 设置密码 requirepass your_redis_password
     ```

### 2.3 容器化部署

#### 2.3.1 Docker 部署

1. **创建 Dockerfile**
   - **前端 Dockerfile**:
     ```dockerfile
     FROM node:14-alpine as build
     WORKDIR /app
     COPY package*.json ./
     RUN npm install
     COPY . .
     RUN npm run build

     FROM nginx:alpine
     COPY --from=build /app/dist /usr/share/nginx/html
     COPY nginx.conf /etc/nginx/conf.d/default.conf
     EXPOSE 80
     CMD ["nginx", "-g", "daemon off;"]
     ```

   - **后端 Dockerfile**:
     ```dockerfile
     FROM node:14-alpine
     WORKDIR /app
     COPY package*.json ./
     RUN npm install
     COPY . .
     RUN npm run build
     EXPOSE 5002
     CMD ["node", "dist/server.js"]
     ```

2. **创建 docker-compose.yml**
   ```yaml
   version: '3'
   services:
     frontend:
       build: ./frontend
       ports:
         - "80:80"
       depends_on:
         - backend

     backend:
       build: ./backend
       ports:
         - "5002:5002"
       depends_on:
         - mongodb
         - redis
       environment:
         - NODE_ENV=production
         - MONGO_URI=mongodb://mongodb:27017/laodong_production
         - REDIS_URL=redis://redis:6379

     mongodb:
       image: mongo:4.4
       volumes:
         - mongodb_data:/data/db
       ports:
         - "27017:27017"

     redis:
       image: redis:6
       volumes:
         - redis_data:/data
       ports:
         - "6379:6379"

   volumes:
     mongodb_data:
     redis_data:
   ```

3. **启动容器**
   ```bash
   docker-compose up -d
   ```

#### 2.3.2 Kubernetes 部署

1. **创建 Kubernetes 配置文件**
   - **前端 Deployment**:
     ```yaml
     apiVersion: apps/v1
     kind: Deployment
     metadata:
       name: laodongzhongcai-frontend
     spec:
       replicas: 2
       selector:
         matchLabels:
           app: laodongzhongcai-frontend
       template:
         metadata:
           labels:
             app: laodongzhongcai-frontend
         spec:
           containers:
           - name: frontend
             image: <frontend-image>
             ports:
             - containerPort: 80
     ```

   - **后端 Deployment**:
     ```yaml
     apiVersion: apps/v1
     kind: Deployment
     metadata:
       name: laodongzhongcai-backend
     spec:
       replicas: 2
       selector:
         matchLabels:
           app: laodongzhongcai-backend
       template:
         metadata:
           labels:
             app: laodongzhongcai-backend
         spec:
           containers:
           - name: backend
             image: <backend-image>
             ports:
             - containerPort: 5002
             env:
             - name: MONGO_URI
               value: mongodb://mongodb:27017/laodong_production
             - name: REDIS_URL
               value: redis://redis:6379
     ```

   - **Service 配置**:
     ```yaml
     apiVersion: v1
     kind: Service
     metadata:
       name: laodongzhongcai-frontend
     spec:
       selector:
         app: laodongzhongcai-frontend
       ports:
       - port: 80
         targetPort: 80
       type: LoadBalancer

     --- 

     apiVersion: v1
     kind: Service
     metadata:
       name: laodongzhongcai-backend
     spec:
       selector:
         app: laodongzhongcai-backend
       ports:
       - port: 5002
         targetPort: 5002
       type: ClusterIP
     ```

2. **应用配置**
   ```bash
   kubectl apply -f frontend-deployment.yaml
   kubectl apply -f backend-deployment.yaml
   kubectl apply -f services.yaml
   ```

## 3. 系统维护

### 3.1 日常维护

#### 3.1.1 日志管理

- **前端日志**:
  - 浏览器控制台日志
  - Nginx 访问日志: `/var/log/nginx/access.log`
  - Nginx 错误日志: `/var/log/nginx/error.log`

- **后端日志**:
  - PM2 日志: `pm2 logs laodongzhongcai-backend`
  - 应用日志: 可配置在 `logs` 目录

- **数据库日志**:
  - MongoDB 日志: `/var/log/mongodb/mongod.log`
  - Redis 日志: `/var/log/redis/redis-server.log`

- **日志轮转**:
  - 配置 logrotate 定期轮转日志
  - 保留 30 天日志
  - 压缩归档旧日志

#### 3.1.2 备份恢复

- **数据备份**:
  - **MongoDB 备份**:
    ```bash
    # 定时备份
    0 2 * * * mongodump --db laodong_production --out /backup/mongodb/$(date +\%Y\%m\%d)
    
    # 备份清理（保留30天）
    0 3 * * * find /backup/mongodb -type d -mtime +30 -exec rm -rf {} \;
    ```

  - **Redis 备份**:
    ```bash
    # 定时备份
    0 2 * * * redis-cli bgsave
    cp /var/lib/redis/dump.rdb /backup/redis/$(date +\%Y\%m\%d).rdb
    
    # 备份清理（保留30天）
    0 3 * * * find /backup/redis -name "*.rdb" -mtime +30 -exec rm -f {} \;
    ```

  - **代码备份**:
    ```bash
    # 定时备份
    0 1 * * * tar -czf /backup/code/laodongzhongcai-$(date +\%Y\%m\%d).tar.gz /opt/laodongzhongcai
    
    # 备份清理（保留30天）
    0 3 * * * find /backup/code -name "*.tar.gz" -mtime +30 -exec rm -f {} \;
    ```

- **数据恢复**:
  - **MongoDB 恢复**:
    ```bash
    mongorestore --db laodong_production /backup/mongodb/20230101/laodong_production
    ```

  - **Redis 恢复**:
    ```bash
    systemctl stop redis-server
    cp /backup/redis/20230101.rdb /var/lib/redis/dump.rdb
    systemctl start redis-server
    ```

### 3.2 故障处理

#### 3.2.1 常见故障

| 故障类型 | 可能原因 | 解决方案 |
|---------|---------|----------|
| 前端页面无法访问 | 服务器宕机、Nginx 配置错误、网络问题 | 检查服务器状态、Nginx 配置、网络连接 |
| 后端 API 无响应 | 服务宕机、数据库连接失败、内存不足 | 检查服务状态、数据库连接、服务器资源 |
| 数据库连接失败 | MongoDB 服务宕机、连接配置错误、网络问题 | 检查 MongoDB 状态、连接配置、网络连接 |
| Redis 连接失败 | Redis 服务宕机、连接配置错误、内存不足 | 检查 Redis 状态、连接配置、内存使用 |
| 登录失败 | 账号密码错误、JWT 配置错误、数据库连接失败 | 检查账号信息、JWT 配置、数据库状态 |
| 文件上传失败 | 文件大小超过限制、权限不足、磁盘空间不足 | 检查文件大小、目录权限、磁盘空间 |
| 系统响应缓慢 | 服务器负载高、数据库查询慢、代码性能问题 | 检查服务器负载、数据库性能、优化代码 |

#### 3.2.2 故障排查流程

1. **确认故障现象**
   - 收集故障发生的时间、具体表现、影响范围
   - 重现故障，确认问题真实性

2. **分析故障原因**
   - 检查相关日志文件
   - 检查服务器状态和资源使用
   - 检查网络连接和服务状态

3. **实施解决方案**
   - 根据分析结果实施相应的解决方案
   - 优先恢复系统功能，再进行优化

4. **验证解决方案**
   - 验证故障是否解决
   - 检查系统是否正常运行
   - 进行相关功能测试

5. **记录故障处理**
   - 记录故障原因、解决方案、处理过程
   - 分析故障发生的根本原因
   - 提出预防措施，避免类似故障再次发生

## 4. 账号管理

### 4.1 账号创建与管理

#### 4.1.1 创建用户账号

系统提供了以下方式创建用户账号：

1. **管理员后台创建**:
   - 登录管理员账号
   - 进入"用户管理"页面
   - 点击"创建用户"按钮
   - 填写用户信息，包括用户名、密码、姓名、电话、角色等
   - 点击"保存"按钮完成创建

2. **API 接口创建**:
   - 使用 `/api/auth/register` 接口
   - 提供必要的用户信息
   - 示例请求:
     ```bash
     curl -X POST http://localhost:5002/api/auth/register \
       -H "Content-Type: application/json" \
       -d '{"username":"newuser","password":"password123","name":"新用户","phone":"13800138000","role":"personal"}'
     ```

#### 4.1.2 账号密码重置

当用户忘记密码时，可通过以下方式重置：

1. **管理员后台重置**:
   - 登录管理员账号
   - 进入"用户管理"页面
   - 找到需要重置密码的用户
   - 点击"重置密码"按钮
   - 设置新密码并保存

2. **邮箱验证重置**:
   - 在登录页面点击"忘记密码"链接
   - 输入注册时的邮箱
   - 系统发送重置链接到邮箱
   - 点击邮件中的链接设置新密码

#### 4.1.3 账号状态管理

- **启用/禁用账号**:
  - 管理员可在"用户管理"页面启用或禁用用户账号
  - 禁用的账号无法登录系统

- **账号锁定**:
  - 系统会在用户连续多次登录失败后自动锁定账号
  - 锁定时间默认为30分钟
  - 管理员可手动解锁被锁定的账号

### 4.2 账号安全最佳实践

#### 4.2.1 密码策略

- **密码强度要求**:
  - 密码长度至少6位
  - 建议使用包含字母、数字和特殊字符的强密码
  - 定期更换密码，建议每90天更换一次

- **密码存储**:
  - 系统使用 bcryptjs 对密码进行加密存储
  - 密码加密后不可逆，即使数据库被泄露也能保障密码安全

#### 4.2.2 登录安全

- **登录尝试限制**:
  - 系统限制每账号每分钟最多5次登录尝试
  - 超过限制后账号会被临时锁定

- **登录日志**:
  - 系统记录所有登录尝试，包括成功和失败的登录
  - 管理员可查看登录日志，及时发现异常登录

#### 4.2.3 权限管理

- **最小权限原则**:
  - 为用户分配最小必要的权限
  - 避免过度授权

- **权限审计**:
  - 定期审计用户权限，确保权限分配合理
  - 及时回收不再需要的权限
